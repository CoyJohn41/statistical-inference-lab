01 — Overview

Project Overview

Most data scientists reach for sklearn immediately. This project takes the opposite approach: every statistical method is derived mathematically before a single line of code is written. The goal is to demonstrate that these aren't black boxes — they're logical consequences of well-understood probability theory.

The toolkit covers simple and multiple linear regression, t-tests, chi-square tests, confidence intervals, and full residual diagnostics. Each implementation is then benchmarked against scipy.stats and sklearn to verify mathematical accuracy.

02 — Mathematical Background

Mathematical Foundation

# Ordinary Least Squares — closed-form solution
β̂ = (XᵀX)⁻¹ Xᵀy

# Variance of estimator
Var(β̂) = σ² (XᵀX)⁻¹

# t-statistic for hypothesis test H₀: βⱼ = 0
t = β̂ⱼ / SE(β̂ⱼ), t ~ t(n - p - 1)

# 95% Confidence Interval
CI = β̂ ± t_{α/2, n-p-1} · SE(β̂)

The OLS estimator is the Best Linear Unbiased Estimator (BLUE) by the Gauss-Markov theorem — provided the classical assumptions hold. This project tests each assumption explicitly using residual analysis.

03 — Implementation

Implementation Approach

  1. Matrix formulation — Express the design matrix X, solve the normal equations (XᵀX)β = Xᵀy using NumPy's linear algebra module.

  2. Coefficient standard errors — Compute residual variance σ² = RSS/(n-p-1), then extract SE from the diagonal of the covariance matrix.

  3. Hypothesis testing — Implement t-tests and F-tests from scratch; compare p-values against scipy.stats output.

  4. Residual diagnostics — Check homoscedasticity (Breusch-Pagan), normality (Shapiro-Wilk), autocorrelation (Durbin-Watson).

  5. Validation layer — Systematically compare every coefficient, SE, and p-value against sklearn/statsmodels output.

import numpy as np from scipy import stats class LinearRegression: """OLS Linear Regression from first principles.""" def fit(self, X, y): # Add intercept column X_b = np.column_stack([np.ones(len(X)), X]) # Normal equations: β = (XᵀX)⁻¹ Xᵀy self.beta_ = np.linalg.solve(X_b.T @ X_b, X_b.T @ y) # Residuals & variance y_hat = X_b @ self.beta_ residuals = y - y_hat n, p = X_b.shape self.sigma2_ = (residuals @ residuals) / (n - p) # Covariance matrix → standard errors cov = self.sigma2_ * np.linalg.inv(X_b.T @ X_b) self.se_ = np.sqrt(np.diag(cov)) # t-statistics and p-values self.t_stats_ = self.beta_ / self.se_ self.p_values_ = 2 * stats.t.sf(np.abs(self.t_stats_), df=n-p) return self def confidence_interval(self, alpha=0.05): # Two-sided CI for each coefficient t_crit = stats.t.ppf(1 - alpha/2, df=len(self.beta_)-1) return np.column_stack([ self.beta_ - t_crit * self.se_, self.beta_ + t_crit * self.se_ ])

05 — Insights

Key Insights

  • The normal equations have no solution when X is rank-deficient — multicollinearity isn't just a statistical problem, it's a linear algebra problem.
  • p-values measure evidence against the null, not effect size. R² measures fit, not causality. These distinctions matter enormously in practice.
  • Heteroscedasticity inflates standard errors unevenly — failing to detect it leads to incorrect inference even when point estimates are unbiased.
  • The matrix formulation generalizes cleanly: polynomial regression, interaction terms, and regularization (Ridge) are all natural extensions.
